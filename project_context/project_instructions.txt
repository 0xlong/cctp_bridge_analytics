3. Key Events to Extract
To build a complete cross-chain record, you must extract and match two specific events using the nonce and sourceDomain fields :

Transfer Phase	Contract	Event & Primary Parameters
Source (Burn)	TokenMessenger	
DepositForBurn(nonce, amount, depositor, mintRecipient, destinationDomain) 

Destination (Mint)	TokenMessenger	
MintAndWithdraw(mintRecipient, amount, mintToken) 

Relay Logic	MessageTransmitter	
MessageReceived(sourceDomain, nonce, sender, messageBody) 

4. Extraction Process (How-To)
Since you are avoiding Dune, use open-source indexing frameworks like Envio or Subsquid to perform the following steps:

Polling: Set up an indexer to poll RPC nodes for logs from the TokenMessenger address across all target chains.

Indexing: Capture every DepositForBurn event on the source chain. Store the nonce, amount, and destinationDomain.

Reconciliation: Monitor the destination chain for a MessageReceived event from the MessageTransmitter that matches the source nonce and sourceDomain.

Completion: Once a match is found, update the record with the destination transaction hash and timestamp to calculate latency.

5. Essential Data Handling
Address Format: CCTP uses bytes32 for all addresses in message payloads to maintain compatibility. You must truncate the leading zeros to convert them back to 20-byte EVM addresses.

Reorg Handling: Ensure your indexer stores the block_hash and log_index to handle chain reorganizations, particularly on lower-latency chains like Polygon.
